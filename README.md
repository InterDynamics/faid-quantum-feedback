## ğŸ› How to Report a Bug

Found a bug? Help us fix it by submitting a detailed bug report!

1. **Check existing issues first:** [Search existing bug reports](https://github.com/InterDynamics/faid-quantum-feedback/issues?q=is%3Aissue+label%3Abug) to see if someone has already reported it
2. **Create a new bug report:** [Submit a bug report](https://github.com/InterDynamics/faid-quantum-feedback/issues/new?assignees=&labels=bug%2Cneeds-triage&template=bug_report.yml&title=%5BBug%5D%3A+) using our template
3. **Provide details:** Include clear reproduction steps, screenshots, and your environment details

**What makes a good bug report:**
- Clear, descriptive title
- Step-by-step reproduction instructions
- Expected vs. actual behavior
- Screenshots or error messages
- Browser and operating system information

## âœ¨ How to Request a Feature

Have an idea to make FAID Quantum better? We'd love to hear it!

1. **Check existing requests first:** [Search existing feature requests](https://github.com/InterDynamics/faid-quantum-feedback/issues?q=is%3Aissue+label%3Aenhancement) to avoid duplicates

**What makes a good feature request:**
- Clear description of the feature

- Priority level and impact on your workflow

## ğŸ¤ Contributing Guidelines

We appreciate your contributions! Here's how to make the most impact:

### Before You Submit

- **Search first:** Always check if your issue or feature request already exists
- **One issue per submission:** Keep bug reports and feature requests focused on a single topic
- **Be respectful:** Follow our code of conduct and be kind to other community members

### Issue Etiquette

- Use clear, descriptive titles
- Fill out all required fields in the templates
- Update your issue if you discover new information
- Close your issue if you've resolved it or found a workaround

### Response Times

- We triage new issues regularly
- Critical bugs are prioritized
- Feature requests are reviewed based on community interest and alignment with our roadmap
